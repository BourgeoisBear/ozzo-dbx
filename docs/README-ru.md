## Описание

ozzo-dbx — это Go-пакет, расширяющий стандартный пакет `database/sql` мощными методами извлечения данных
и DB-agnostic (независимым от БД) построителем запросов. ozzo-dbx не является ORM. Пакет обладает следующими возможностями:

* Заполнение данных в структуры и NullString-карты
* Именованное связывание параметров
* DB-agnostic методы построения запросов, включая SELECT, запросы для манипуляции с данными и запросы для работы со схемой БД
* Вставка, обновление и удаление модельных структур
* Мощный построитель условий запроса
* Открытая архитектура, позволяющая добавлять поддержку новых баз данных или настраивать существующую
* Логирование выполненных SQL-запросов
* Поддержка основных реляционных баз данных

Пример использования библиотеки в реальном приложении — проект [go-rest-api](https://github.com/qiangxue/go-rest-api), представляющий собой стартовый набор для создания RESTful API на Go.

## Требования

Go 1.21 или выше.

## Установка

Выполните следующую команду для установки пакета:

```
go get github.com/go-ozzo/ozzo-dbx
```

Дополнительно установите пакет драйвера для используемой базы данных. Полный список доступен на странице
[SQL database drivers](https://github.com/golang/go/wiki/SQLDrivers). Например, при работе с MySQL
установите следующий пакет:

```
go get github.com/go-sql-driver/mysql
```

и импортируйте его в основном коде:

```go
import _ "github.com/go-sql-driver/mysql"
```

## Поддерживаемые базы данных

Из коробки полностью поддерживаются следующие базы данных:

* SQLite (ключ драйвера `"sqlite3"` для [mattn/go-sqlite3](https://github.com/mattn/go-sqlite3) или `"sqlite"` для [modernc.org/sqlite](https://pkg.go.dev/modernc.org/sqlite) — драйвер без CGo)
* MySQL
* PostgreSQL
* MS SQL Server (2012 или выше)
* Oracle

Операции со схемой SQLite `DropColumn`, `RenameColumn` и `RenameTable` поддерживаются через стандартный синтаксис `ALTER TABLE` (требуется SQLite 3.25.0+ для переименования, 3.35.0+ для удаления столбца).

Для остальных баз данных построитель запросов может работать некорректно. Вы можете создать собственный построитель для решения этой проблемы — подробности описаны в последнем разделе.

## С чего начать

Приведённый ниже код демонстрирует использование пакета для доступа к данным базы MySQL.

```go
import (
	"fmt"
	"github.com/go-ozzo/ozzo-dbx"
	_ "github.com/go-sql-driver/mysql"
)

func main() {
	db, _ := dbx.Open("mysql", "user:pass@/example")

	// создаём новый запрос
	q := db.NewQuery("SELECT id, name FROM users LIMIT 10")

	// извлекаем все строки в срез структур
	var users []struct {
		ID, Name string
	}
	err := q.All(&users)

	// извлекаем одну строку в структуру
	var user struct {
		ID, Name string
	}
	err = q.One(&user)

	// извлекаем одну строку в строковую карту
	data := dbx.NullStringMap{}
	err = q.One(data)

	// извлечение строка за строкой
	rows2, _ := q.Rows()
	for rows2.Next() {
		_ = rows2.ScanStruct(&user)
		// rows.ScanMap(data)
		// rows.Scan(&id, &name)
	}
}
```

Следующий пример показывает, как использовать возможности построителя запросов.

```go
import (
	"fmt"
	"github.com/go-ozzo/ozzo-dbx"
	_ "github.com/go-sql-driver/mysql"
)

func main() {
	db, _ := dbx.Open("mysql", "user:pass@/example")

	// строим SELECT-запрос
	//   SELECT `id`, `name` FROM `users` WHERE `name` LIKE '%Charles%' ORDER BY `id`
	q := db.Select("id", "name").
		From("users").
		Where(dbx.Like("name", "Charles")).
		OrderBy("id")

	// извлекаем все строки в срез структур
	var users []struct {
		ID, Name string
	}
	err := q.All(&users)

	// строим INSERT-запрос
	//   INSERT INTO `users` (`name`) VALUES ('James')
	err = db.Insert("users", dbx.Params{
		"name": "James",
	}).Execute()
}
```

## Соединение с базой данных

Для подключения к базе данных используйте `dbx.Open()` аналогично методу `Open()` из пакета `database/sql`.

```go
db, err := dbx.Open("mysql", "user:pass@hostname/db_name")
```

Метод возвращает экземпляр `dbx.DB`, который используется для создания и выполнения запросов к БД. Обратите внимание,
что реальное соединение устанавливается только при первом запросе через возвращённый экземпляр `dbx.DB`. Метод также
не проверяет корректность строки подключения. Используйте `dbx.MustOpen()`, чтобы убедиться в правильности
строки подключения.

## Выполнение запросов

Для выполнения SQL-запроса сначала создайте экземпляр `dbx.Query` через `DB.NewQuery()`, передав
SQL-выражение. Затем вызовите `Query.Execute()`, если запрос не предназначен для извлечения данных.
Например:

```go
q := db.NewQuery("UPDATE users SET status=1 WHERE id=100")
result, err := q.Execute()
```

Если SQL-запрос возвращает данные (например, SELECT), следует вызвать один из перечисленных ниже методов,
которые выполнят запрос и сохранят результат в указанные переменные.

* `Query.All()`: заполняет все строки результата в срез структур или `NullString`-карт. Также поддерживаются срезы указателей (`[]*Struct`).
* `Query.One()`: сохраняет первую строку результата в структуру или `NullString`-карту.
* `Query.Column()`: заполняет первый столбец результата в срез.
* `Query.Row()`: заполняет первую строку результата в список переменных — по одной на каждый возвращаемый столбец.
* `Query.Rows()`: возвращает экземпляр `dbx.Rows` для построчного извлечения данных.

Например:

```go
type User struct {
	ID   int
	Name string
}

var (
	users []User
	user User

	row dbx.NullStringMap

	id   int
	name string

	err error
)

q := db.NewQuery("SELECT id, name FROM users LIMIT 10")

// заполняем срез User всеми строками результата
err = q.All(&users)
fmt.Println(users[0].ID, users[0].Name)

// заполняем структуру User первой строкой результата
err = q.One(&user)
fmt.Println(user.ID, user.Name)

// сохраняем первую строку в NullString-карту
err = q.One(&row)
fmt.Println(row["id"], row["name"])

var ids []int
err = q.Column(&ids)
fmt.Println(ids)

// заполняем переменные id и name первой строкой результата
err = q.Row(&id, &name)

// извлекаем данные строка за строкой
rows, _ := q.Rows()
for rows.Next() {
	_ = rows.ScanMap(&row)
}
```

При заполнении структуры применяются следующие правила для определения соответствия столбцов полям структуры:

* Заполняются только экспортируемые поля.
* Поле получает данные, если его имя отображается на столбец в соответствии с функцией `Query.FieldMapper`.
  По умолчанию функция разделяет слова в имени поля символом подчёркивания и переводит их в нижний регистр.
  Например, поле `FirstName` отображается в столбец `first_name`, а `MyID` — в `my_id`.
* Если поле имеет тег `db`, значение тега используется как имя соответствующего столбца. Если тег `db` равен `-`,
  поле заполняться не будет.
* Анонимные поля типа структуры раскрываются, и их поля заполняются по описанным выше правилам.
* Именованные поля типа структуры также раскрываются, но их вложенные поля при заполнении получают префикс
  в виде имени родительской структуры.

Исключение из правил раскрытия: если тип структуры реализует интерфейс `sql.Scanner` или является типом `time.Time`,
поле заполняется целиком драйвером БД. Если поле является указателем на некоторый тип,
для него будет выделена память, и оно будет заполнено результатом запроса, если значение не null.

Приведённый ниже пример показывает, как поля заполняются по описанным правилам:

```go
type User struct {
	id     int
	Type   int `db:"-"`
	MyName string `db:"name"`
	Profile
	Address Address `db:"addr"`
}

type Profile struct {
	Age int
}

type Address struct {
	City string
}
```

* `User.id`: не заполняется, поскольку поле неэкспортируемое;
* `User.Type`: не заполняется, так как тег `db` равен `-`;
* `User.MyName`: заполняется из столбца `name` в соответствии с тегом `db`;
* `Profile.Age`: заполняется из столбца `age`, так как `Profile` — анонимное поле;
* `Address.City`: заполняется из столбца `addr.city`, так как `Address` — именованное поле типа структуры
  и его вложенные поля получают префикс `addr.` в соответствии с тегом `db`.

Обратите внимание: если столбец результата не имеет соответствующего поля в структуре, он игнорируется.
Аналогично, если поле структуры не имеет соответствующего столбца в результате, оно не заполняется.

## Связывание параметров

SQL-запрос, как правило, параметризуется динамическими значениями. Например, вы можете запросить запись пользователя
по его ID, полученному от клиента. В таких случаях используется связывание параметров — это практически
всегда предпочтительно для защиты от SQL-инъекций. В отличие от `database/sql`, где применяется анонимное связывание,
`ozzo-dbx` использует именованное связывание параметров. *Анонимное связывание не поддерживается*, поскольку оно
конфликтует с именованными параметрами. Например:

```go
q := db.NewQuery("SELECT id, name FROM users WHERE id={:id}")
q.Bind(dbx.Params{"id": 100})
err := q.One(&user)
```

В приведённом примере будет выбрана запись пользователя с `id` равным 100. Метод `Query.Bind()` связывает набор
именованных параметров с SQL-запросом, содержащим плейсхолдеры в формате `{:ParamName}`.

Если один и тот же SQL-запрос нужно выполнить несколько раз с разными значениями параметров, его можно
подготовить для повышения производительности:

```go
q := db.NewQuery("SELECT id, name FROM users WHERE id={:id}")
q.Prepare()
defer q.Close()

q.Bind(dbx.Params{"id": 100})
err := q.One(&user)

q.Bind(dbx.Params{"id": 200})
err = q.One(&user)

// ...
```


## Отменяемые запросы

Запросы поддерживают отмену при использовании с `context.Context`. Вызвав `Query.WithContext()`, вы можете
связать контекст с запросом и использовать его для отмены запроса во время выполнения. Например:

```go
q := db.NewQuery("SELECT id, name FROM users")
err := q.WithContext(ctx).All(&users)
```


## Построение запросов

Вместо написания SQL-запросов вручную `ozzo-dbx` позволяет строить их программно, что ведёт к более чистому,
безопасному и независимому от БД коду. Доступны три типа запросов: SELECT-запросы, запросы для манипуляции
с данными и запросы для работы со схемой БД.

### Построение SELECT запросов

Построение SELECT-запроса начинается с вызова `DB.Select()`. Различные части запроса формируются
соответствующими методами построителя. Например:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")
err := db.Select("id", "name").
	From("users").
	Where(dbx.HashExp{"id": 100}).
	One(&user)
```

Приведённый код сгенерирует и выполнит следующий SQL-запрос:

```sql
SELECT `id`, `name` FROM `users` WHERE `id`={:p0} 
```

Обратите внимание, что имена таблиц и столбцов автоматически экранируются в соответствии с текущей СУБД.
Для значения `p0` в условии `WHERE` используется связывание параметров.

Каждое ключевое слово SQL имеет соответствующий метод построителя. Например, `SELECT` — `Select()`,
`FROM` — `From()`, `WHERE` — `Where()` и так далее. Эти методы можно объединять в цепочку,
как при написании обычного SQL. Каждый метод возвращает экземпляр запроса (типа `dbx.SelectQuery`).
После завершения построения запроса вызовите `One()`, `All()` и другие методы для выполнения запроса
и заполнения данных. Также можно явно вызвать `Build()` для преобразования в экземпляр `dbx.Query`,
что даст доступ к сформированному SQL-выражению.


### Построение условий запроса

`ozzo-dbx` поддерживает гибкий и мощный построитель условий, который используется в SQL-выражениях
`WHERE`, `HAVING` и других. Например:

```go
// id=100
dbx.NewExp("id={:id}", dbx.Params{"id": 100})

// id=100 AND status=1
dbx.HashExp{"id": 100, "status": 1}

// status=1 OR age>30
dbx.Or(dbx.HashExp{"status": 1}, dbx.NewExp("age>30"))

// name LIKE '%admin%' AND name LIKE '%example%'
dbx.Like("name", "admin", "example")
```

При построении выражений условий значения параметров подставляются через связывание, что предотвращает
SQL-инъекции. Имена столбцов в выражениях автоматически экранируются.
Доступны следующие функции для построения условий:

* `dbx.NewExp()`: создаёт условие по заданной строке выражения и связываемым параметрам. Например,
`dbx.NewExp("id={:id}", dbx.Params{"id":100})` создаст выражение `id=100`.
* `dbx.HashExp`: тип-карта, представляющий пары имя-значение, объединённые оператором `AND`. Например,
`dbx.HashExp{"id":100, "status":1}` создаст `id=100 AND status=1`.
* `dbx.Not()`: создаёт выражение `NOT`, добавляя `NOT` перед заданным выражением.
* `dbx.And()`: создаёт выражение `AND`, объединяя заданные выражения оператором `AND`.
* `dbx.Or()`: создаёт выражение `OR`, объединяя заданные выражения оператором `OR`.
* `dbx.In()`: создаёт выражение `IN` для указанного столбца и диапазона значений.
Например, `dbx.In("age", 30, 40, 50)` создаст выражение `age IN (30, 40, 50)`.
Если диапазон пуст, генерируется выражение, представляющее ложное значение.
* `dbx.NotIn()`: создаёт выражение `NOT IN`. Аналогично `dbx.In()`.
* `dbx.Like()`: создаёт выражение `LIKE` для указанного столбца и диапазона значений. Например,
`dbx.Like("title", "golang", "framework")` создаст выражение `title LIKE "%golang%" AND title LIKE "%framework%"`.
Полученное выражение можно дополнительно настроить вызовом `Escape()` и/или `Match()`.
Если диапазон значений пуст, генерируется пустое выражение.
* `dbx.NotLike()`: создаёт выражение `NOT LIKE`. Аналогично `dbx.Like()`.
* `dbx.OrLike()`: создаёт выражение `LIKE`, но объединяет подвыражения через `OR` вместо `AND`.
* `dbx.OrNotLike()`: создаёт выражение `NOT LIKE`, объединяя подвыражения через `OR` вместо `AND`.
* `dbx.Exists()`: создаёт выражение `EXISTS`, добавляя `EXISTS` перед заданным выражением.
* `dbx.NotExists()`: создаёт выражение `NOT EXISTS`, добавляя `NOT EXISTS` перед заданным выражением.
* `dbx.Between()`: создаёт выражение `BETWEEN`. Например, `dbx.Between("age", 30, 40)` создаст
выражение `age BETWEEN 30 AND 40`.
* `dbx.NotBetween()`: создаёт выражение `NOT BETWEEN`.

Вы также можете создавать собственные вспомогательные функции для построения условий — достаточно, чтобы
функция возвращала объект, реализующий интерфейс `dbx.Expression`.
 

### Построение запросов для манипуляции с данными

Запросы манипуляции с данными изменяют данные в базе: INSERT, UPDATE, DELETE.
Они создаются вызовом соответствующих методов `DB`. Например:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")

// INSERT INTO `users` (`name`, `email`) VALUES ({:p0}, {:p1})
err := db.Insert("users", dbx.Params{
	"name": "James",
	"email": "james@example.com",
}).Execute()

// UPDATE `users` SET `status`={:p0} WHERE `id`={:p1}
err = db.Update("users", dbx.Params{"status": 1}, dbx.HashExp{"id": 100}).Execute()

// DELETE FROM `users` WHERE `status`={:p0}
err = db.Delete("users", dbx.HashExp{"status": 2}).Execute()
```

При построении запросов для манипуляции с данными не забудьте вызвать `Execute()` для их выполнения.

### Построение запросов для работы со схемой БД

Запросы для работы со схемой изменяют структуру базы данных: создание таблиц, добавление столбцов и т.д.
Они создаются вызовом соответствующих методов `DB`. Например:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")

// CREATE TABLE `users` (`id` int primary key, `name` varchar(255))
q := db.CreateTable("users", map[string]string{
	"id": "int primary key",
	"name": "varchar(255)",
})
err := q.Execute()
```

## CRUD-операции

Хотя ozzo-dbx не является ORM, пакет предоставляет удобный способ выполнения типичных CRUD-операций
(Create, Read, Update, Delete) без написания SQL-запросов вручную.

Для использования CRUD определите тип структуры для таблицы. По умолчанию структура связывается с таблицей,
имя которой соответствует snake_case-версии имени типа. Например, структура `MyCustomer`
соответствует таблице `my_customer`. Имя таблицы можно задать явно, реализовав
интерфейс `dbx.TableModel`:

```go
type MyCustomer struct{}

func (c MyCustomer) TableName() string {
	return "customer"
}
```

Обратите внимание: метод `TableName` должен быть определён с получателем по значению, а не по указателю.

Если структура содержит поле `ID` или `Id`, по умолчанию оно считается первичным ключом.
Для использования другого поля в качестве первичного ключа укажите тег `db:"pk"`. Для составных ключей
можно пометить несколько полей. Если нужно также задать имя столбца для поля первичного ключа,
используйте формат тега `db:"pk,col_name"`.

Добавить общий префикс или суффикс к именам таблиц можно через `DB.TableMapFunc`. Например,
следующий код добавляет префикс `tbl_` ко всем именам таблиц:

```go
db.TableMapper = func(a interface{}) string {
    return "tbl_" + GetTableName(a)
}
```

### Создание (Create)

Для создания (вставки) новой строки через модель вызовите метод `ModelQuery.Insert()`:

```go
type Customer struct {
	ID     int
	Name   string
	Email  string
	Status int
}

db, _ := dbx.Open("mysql", "user:pass@/example")

customer := Customer{
	Name: "example",
	Email: "test@example.com",
}
// INSERT INTO customer (name, email, status) VALUES ('example', 'test@example.com', 0)
err := db.Model(&customer).Insert()
```

Будет вставлена строка со значениями *всех* публичных полей структуры (кроме пустого поля первичного ключа).
Если поле первичного ключа содержит нулевое значение (целочисленный ноль или nil-указатель), оно считается
автоинкрементным и после успешной вставки автоматически заполняется идентификатором последней вставленной записи.

Вы можете явно указать поля для вставки, передав их имена в `Insert()`.
Также можно исключить определённые поля вызовом `Exclude()` перед `Insert()`:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")

// вставить только поля Name и Email
err := db.Model(&customer).Insert("Name", "Email")
// вставить все публичные поля кроме Status
err = db.Model(&customer).Exclude("Status").Insert()
// вставить только Name
err = db.Model(&customer).Exclude("Status").Insert("Name", "Status")
```

### Чтение (Read)

Для чтения модели по первичному ключу вызовите `SelectQuery.Model()`:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")

var customer Customer
// SELECT * FROM customer WHERE id=100
err := db.Select().Model(100, &customer)

// SELECT name, email FROM customer WHERE status=1 AND id=100
err = db.Select("name", "email").Where(dbx.HashExp{"status": 1}).Model(100, &customer)
```

Метод `SelectQuery.Model()` не поддерживает составные первичные ключи. В этом случае используйте `SelectQuery.One()`:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")

var orderItem OrderItem

// SELECT * FROM order_item WHERE order_id=100 AND item_id=20
err := db.Select().Where(dbx.HashExp{"order_id": 100, "item_id": 20}).One(&orderItem)
```

В приведённых запросах метод `From()` не вызывается — запрос автоматически определяет таблицу по заполняемой
модельной структуре. Если структура реализует `TableModel`, используется значение `TableName()`.
В противном случае именем таблицы будет snake_case-версия имени типа структуры.

Для получения списка моделей вызовите `SelectQuery.All()`. Аналогично, `From()` указывать не нужно,
если имя таблицы выводится из модельной структуры.


### Обновление (Update)

Для обновления модели вызовите `ModelQuery.Update()`. Как и `Insert()`, по умолчанию `Update()` обновляет
*все* публичные поля кроме полей первичного ключа. Включение и исключение полей работает так же, как
описано для `Insert()`:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")

// обновить все публичные поля customer
err := db.Model(&customer).Update()
// обновить только Status
err = db.Model(&customer).Update("Status")
// обновить все публичные поля кроме Status
err = db.Model(&customer).Exclude("Status").Update()
```

Метод `Update()` считает первичные ключи неизменяемыми и использует их значения для поиска обновляемой строки.
Если модель не имеет первичного ключа, будет возвращена ошибка.


### Удаление (Delete)

Для удаления модели вызовите `ModelQuery.Delete()`. Метод удаляет строку по значению первичного ключа модели.
Если модель не имеет первичного ключа, будет возвращена ошибка:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")

err := db.Model(&customer).Delete()
```

### Работа с NULL

Для представления nullable-значения базы данных можно использовать тип-указатель. Если указатель равен nil,
соответствующее значение в базе — null.

Другой вариант — типы `sql.NullXyz`. Например, для nullable строкового столбца используйте `sql.NullString`.
Поле `NullString.Valid` указывает, является ли значение null, а `NullString.String` содержит строковое значение.
Поскольку типы `sql.NullXyz` не поддерживают маршалинг JSON, вместо них можно использовать пакет
[null](https://github.com/guregu/null).

Пример работы с null-значениями:

```go
type Customer struct {
	ID        int
	Email     string
	FirstName *string        // указатель для представления null
	LastName  sql.NullString // sql.NullString для представления null
}
```

## Экранирование имён таблиц и столбцов

Различные СУБД по-разному экранируют имена таблиц и столбцов. Для написания DB-agnostic SQL-запросов ozzo-dbx
использует специальный синтаксис. Слово, заключённое в `{{` и `}}`, трактуется как имя таблицы и экранируется
в соответствии с текущим драйвером БД. Аналогично, слово в `[[` и `]]` трактуется как имя столбца.
Например, при работе с MySQL следующий запрос будет корректно экранирован:

```go
// SELECT * FROM `users` WHERE `status`=1
q := db.NewQuery("SELECT * FROM {{users}} WHERE [[status]]=1")
```

Обратите внимание: если имя таблицы или столбца содержит префикс, оно по-прежнему будет правильно экранировано.
Например, `{{public.users}}` для PostgreSQL будет экранировано как `"public"."users"`.

## Использование транзакций

Все описанные выше методы выполнения и построения запросов работают с транзакциями:

```go
db, _ := dbx.Open("mysql", "user:pass@/example")

tx, _ := db.Begin()

_, err1 := tx.Insert("users", dbx.Params{
	"name": "user1",
}).Execute()
_, err2 := tx.Insert("users", dbx.Params{
	"name": "user2",
}).Execute()

if err1 == nil && err2 == nil {
	tx.Commit()
} else {
	tx.Rollback()
}
```

Метод `DB.Transactional()` упрощает работу с транзакциями, избавляя от необходимости явно
вызывать commit или rollback. Он запускает транзакцию и автоматически откатывает её, если callback
возвращает ошибку. В противном случае транзакция автоматически фиксируется.


```go
db, _ := dbx.Open("mysql", "user:pass@/example")

err := db.Transactional(func(tx *dbx.Tx) error {
	var err error
	_, err = tx.Insert("users", dbx.Params{
		"name": "user1",
	}).Execute()
	if err != nil {
		return err
	}
	_, err = tx.Insert("users", dbx.Params{
		"name": "user2",
	}).Execute()
	return err
})

fmt.Println(err)
```

## Логирование выполненных SQL-запросов

Для логирования и инструментирования запросов к БД можно установить логгеры для соединения. Доступны три вида логгеров:

* `DB.LogFunc`: вызывается при каждом выполнении SQL-запроса. Сигнатура функции совпадает с `fmt.Printf`,
  что делает использование максимально простым.
* `DB.QueryLogFunc`: вызывается при каждом запросе данных (SELECT и т.п.).
* `DB.ExecLogFunc`: вызывается при выполнении SQL-выражения (INSERT, UPDATE, DELETE и т.п.).

Следующий пример демонстрирует использование логгеров:

```go
import (
	"fmt"
	"log"
	"github.com/go-ozzo/ozzo-dbx"
)

func main() {
	db, _ := dbx.Open("mysql", "user:pass@/example")

	// простое логирование
	db.LogFunc = log.Printf

	// или более гибкое логирование
	db.QueryLogFunc = func(ctx context.Context, t time.Duration, sql string, rows *sql.Rows, err error) {
		log.Printf("[%.2fms] Query SQL: %v", float64(t.Milliseconds()), sql))
	}
	db.ExecLogFunc = func(ctx context.Context, t time.Duration, sql string, result sql.Result, err error) {
		log.Printf("[%.2fms] Execute SQL: %v", float64(t.Milliseconds()), sql))
	}
	// ...
)
``` 

## Поддержка новых баз данных

Несмотря на то, что `ozzo-dbx` из коробки поддерживает большинство основных реляционных баз данных, открытая архитектура
позволяет добавлять поддержку новых СУБД. Для этого необходимо:

* Создать структуру, реализующую интерфейс `QueryBuilder`. Можно использовать `BaseQueryBuilder` напрямую
  или расширить его через композицию.
* Создать структуру, реализующую интерфейс `Builder`. Можно расширить `BaseBuilder` через композицию.
* Написать функцию `init()` для регистрации нового построителя в `dbx.BuilderFuncMap`.
